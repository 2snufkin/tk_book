PDF Generation in the Tumorotek Application

Overview​

The Tumorotek application generates PDF documents dynamically by structuring content in XML and transforming it into a formatted PDF using XSL-FO (Apache FOP). This process is typically triggered by user actions (e.g., clicking a "Print" button) and allows customization through modal dialogs (e.g., headers/footers).

The workflow consists of three main stages:

  1. XML Document Creation – A structured JDOM XML is generated, representing the PDF’s content.
  2. XSLT Transformation – The XML is converted into XSL-FO (Formatting Objects) using predefined XSLT templates.
  3. PDF Rendering – Apache FOP processes the XSL-FO to produce the final PDF, which is then downloaded by the user.

This document explains each step in detail, including the XML schema, transformation logic, and key components involved.

Key Steps in PDF Generation​

1. User Interaction & Customization​

  • Trigger: A user action (e.g., clicking "Print") invokes methods like onClick$print(), which may open a modal for header/footer input.
  • Customization: Users can define headers, footers, and other dynamic content (e.g., dates, logos) before generation.

2. XML Structure Creation​

The system constructs a JDOM XML document with the following structure:

Core Elements​

ElementDescriptionExample Usage
<Impression>Root element containing all pages.<Impression><Page>...</Page></Impression>
<Page>Represents a PDF page with a title and content.<Page><Titre>Bank Details</Titre>...</Page>
<Paragraphe>A content block with key-value pairs (<CoupleValeur>).Used for structured text (e.g., "Bank Name: XYZ").
<Liste>A table with headers (<EnteteListe>) and rows (<LigneListe>).Displays tabular data (e.g., container lists).
<HautDePage> / <BasDePage>Custom headers/footers.<HautDePage>Confidential Report</HautDePage>

Example XML Construction (Java Code)​

// Create the root XML document  
Document document = ManagerLocator.getXmlUtils().createJDomDocument();  
Element root = document.getRootElement();  

// Add header/footer  
ManagerLocator.getXmlUtils().addHautDePage(root, "Custom Header", false, null);  
ManagerLocator.getXmlUtils().addBasDePage(root, "Footer with date: " + new Date());  

// Add a page with a table of containers  
EnteteListe headers = new EnteteListe(new String[] {"Code", "Name", "Temperature"});  
LigneListe[] rows = new LigneListe[containers.size()];  
ListeElement table = new ListeElement(null, headers, rows);  
ManagerLocator.getXmlUtils().addListe(root, table);

3. XSLT Transformation to XSL-FO​

The XML is converted into XSL-FO using an XSLT stylesheet (e.g., xslPdfMainResource.xsl).

Key XSLT Mappings​

XML ElementXSL-FO Equivalent
<Paragraphe><fo:block>
<Liste><fo:table>
<CoupleValeur><fo:inline> (key-value pairs)
<HautDePage><fo:static-content flow-name="header">

Example XSLT Snippet​

<xsl:template match="Paragraphe">  
  <fo:block font-size="12pt">  
    <xsl:value-of select="TitreParagraphe"/>  
  </fo:block>  
</xsl:template>

4. PDF Rendering with Apache FOP​

The XSL-FO document is processed into a PDF byte array:

Steps in PDF Generation​

  1. Initialize FOP: 
    FopFactory fopFactory = FopFactory.newInstance();  
    FOUserAgent foUserAgent = fopFactory.newFOUserAgent();
  2. Transform XSL-FO to PDF: 
    Fop fop = fopFactory.newFop(MimeConstants.MIME_PDF, foUserAgent, out);  
    Transformer transformer = TransformerFactory.newInstance().newTransformer();  
    transformer.transform(new DOMSource(transformedXml), new SAXResult(fop.getDefaultHandler()));
  3. Output PDF: The resulting byte array is sent to the user via Filedownload.save().

Specialized Use Cases​

1. Dynamic Tables (e.g., Container Lists)​

  • XML Structure: 
    <Liste>  
      <EnteteListe>  
        <NomColonne>Code</NomColonne>  
        <NomColonne>Temperature</NomColonne>  
      </EnteteListe>  
      <LigneListe>  
        <Cellule>A1</Cellule>  
        <Cellule>-80°C</Cellule>  
      </LigneListe>  
    </Liste>
  • XSL-FO Output: 
    <fo:table>  
      <fo:table-header>  
        <fo:table-cell><fo:block>Code</fo:block></fo:table-cell>  
      </fo:table-header>  
      <fo:table-body>  
        <fo:table-row>  
          <fo:table-cell><fo:block>A1</fo:block></fo:table-cell>  
        </fo:table-row>  
      </fo:table-body>  
    </fo:table>

2. Multi-Page Documents​

  • XML Structure: 
    <Impression>  
      <Page><Titre>Page 1</Titre>...</Page>  
      <Page><Titre>Page 2</Titre>...</Page>  
    </Impression>

FAQ​

Q: How are images (e.g., logos) included?
→ Images are referenced in XML (e.g., adrImages + "logo.png") and rendered via <fo:external-graphic>.

Q: Can the PDF layout be customized?
→ Yes, by modifying the XSLT templates (e.g., adjusting fonts, margins, or table styles).

Q: How are empty fields handled?
→ Conditional logic in Java omits empty sections (e.g., displaying "-" for missing data).

Conclusion​

The Tumorotek PDF generation system leverages XML and XSL-FO to create structured, customizable documents. By separating content (XML) from styling (XSLT), the system ensures flexibility and maintainability. Future enhancements could include dynamic template selection or advanced formatting options.

For further details, refer to the XmlUtilsImpl and XSLTransformer classes in the codebase.

Best Practices​

  1. Error Handling

    • Always wrap PDF generation in try-catch blocks
    • Provide user feedback for failures
    • Clean up resources in finally blocks

  2. Content Organization

    • Split content generation into logical methods
    • Use meaningful method names (e.g., addTable(), addSummarySection())
    • Pass page elements as parameters for content addition

  3. User Experience

    • Show loading indicators during generation
    • Provide clear feedback on success/failure
    • Use meaningful filenames with timestamps

Utility Methods​

The XmlUtils class provides several helpful methods:

  • createJDomDocument(): Creates empty document
  • addBasDePage(): Adds footer
  • addHautDePage(): Adds header
  • addPage(): Creates new page
  • creerPdf(): Converts document to PDF bytes

Example Content Section​

private void addContentSections(Element page) {
    addTable(page);
    addSummary(page);
    addMetadata(page);
}

Note: Customize the filename prefix, headers, and footers according to your application's needs. Always test PDF generation with various content types and sizes to ensure proper formatting.
